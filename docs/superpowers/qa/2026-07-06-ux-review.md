# berth-app — Dev-UX Review (2026-07-06)

Method: hands-on preview pass with fixture agents + accumulated QA/live-harness
knowledge, judged against the dev jobs-to-be-done: kick off work fast → monitor
many agents → intervene when blocked → review output → land it → iterate.

## Verdict

The plumbing is solid (spawn/attach/steer/diff/PR all work E2E) but the app is
still **organized around containers, not around the dev's attention**. The two
highest-leverage changes are making "needs you" one keystroke away end-to-end,
and making the sidebar carry enough context to triage without clicking.

## P1 — highest leverage

1. **Blocked-agent fast path.** Today: notice 🟡 → click agent → Terminal →
   read prompt → type. The blocked *reason* lives only in a tooltip and the
   tray. Add: reason as a second line on the sidebar row + a "Respond" banner
   inside the workspace when State=blocked that focuses the terminal with one
   keystroke; `⌘1..9` to jump to the Nth agent; tray/notification click →
   terminal (not just app focus).
2. **Sidebar information density.** Rows show name+type only. Devs juggling
   3 tasks on 2 repos need: repo short-name (second line), elapsed runtime,
   blocked reason. One glance = full triage.
3. **Keyboard-first operation.** No j/k/enter in sidebar, no ⌘T terminal /
   ⌘D diff tab switching, palette lacks per-agent verbs (steer, stop, clone,
   diff of selected). The TUI is more keyboard-efficient than the GUI today.
4. **First-run/empty state.** "Select an agent" dead pane on launch. Replace
   with: spawn CTA, saved projects, keyboard cheat-sheet. New-user path into
   saved-repo chips is invisible until one exists.

## P2 — real friction, medium cost

5. **Prompt visibility.** An agent's original prompt/task is nowhere in the
   UI (label exists engine-side). Show it in Info + as row tooltip; devs
   forget which "fix-ci" was which.
6. **Output tab is last-message-only.** No changed-files list, no commits
   (spec cut). For "is it done?" triage those two lists matter more than the
   message. `output --json` already computes them engine-side… only the
   3-key JSON is exposed — small engine addition, big triage win.
7. **Terminal QoL:** no ⌘F search (xterm search addon is installed but not
   wired), no font-size control, no "copy last output" button, split limited
   to manual dropdown (no "split with clone").
8. **Diff tab:** no per-file stage/revert (all-or-nothing), no side-by-side
   toggle (engine has `--side-by-side`), checkpoints are a raw `pre` dump
   with a free-text ref field — should be clickable rows with a Restore
   button each.
9. **Steer ergonomics:** single-line input, no history, no multi-line paste;
   Enter doesn't submit. Devs will steer constantly; make it a real mini
   composer (↑ history, ⌘Enter submit).

## P3 — worth doing, not urgent

10. **Session timeline per agent** — audit slice exists in Info; promote to a
    mini activity feed (spawned → checkpoint → steered → blocked → done)
    instead of raw timestamps.
11. **Cost view is a text dump** — even a per-agent table with totals beats
    the `pre`; per-model chart only if history grows.
12. **Live sidebar stats staleness** (deliberate v1 tradeoff): CPU/mem only
    refresh on state change. Add a slow (30s) heartbeat merge or drop the
    numbers from Info until fresh.
13. **Fleet view** cannot stop/retry a whole fleet, and pipeline runs give no
    feedback until containers appear (~5-10s dead air) — optimistic "starting…"
    row would fix the uncertainty.
14. **Window title** never changes — show selected agent + needs-you count
    (helps ⌘Tab).
15. **Light mode / system appearance** — hardcoded dark; fine personally,
    cheap to respect `prefers-color-scheme` since Tailwind is set up.

## Deliberate non-goals (stay lazy)

- Session transcript browser in-app (use `berth sessions`/replay; big lift).
- Editor embedding, file browser (open-in-editor action would cover 90%:
  one `open` command on a checked-out worktree — but workspaces live inside
  the VM, so it's actually a handoff feature → engine `handoff --to-local`
  exists; a "Handoff to local" quick action is the lazy bridge).
- Multi-window, theming gallery, onboarding tours.

## Suggested order (if implementing)

P1.1 blocked fast-path → P1.2 sidebar density → P2.6 files/commits in Output
→ P1.3 keyboard nav → P2.9 steer composer → P2.8 diff per-file + side-by-side
→ rest opportunistically.
